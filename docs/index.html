1-3. 既存AIの限界は？

  • 計算の限界：

    • 決定性チューリングマシンと近似可能な計算機では、ある種の問題を現実的な時間で解くことが不可能である
    • いかなる計算機であっても、明確に解けない問題が存在する

  • ディープラーニングの役割：

    • 基本的には入力と出力の関係性を自動で構成 (近似) するだけである $\hat{f}: x \to y$
    • ここで、関係性を自動で構成するだけでは、人間に近いレベルの「思考」をしているとは言えない
      • タスクの学習も、あくまで制約条件化での最適化でしかない (制約条件は人間が設定している)
    • 実は、実用的なAIのほとんどは、人間のアイデア/実装による部分の寄与が大きい

とはいえ、上記を以ってしても、AIで自動化できるタスクは多岐に渡ります。 また、問題の適当な近似を考えて、計算を単純化しAIに担うということも十分に有効です。

個人的な予想としては、人間の知性を真に凌駕するAIが登場するには、 計算機のさらなる発展と生命科学的なアプローチがより必要となってくると考えています。 (e.g. 分子コンピュータ，遺伝的アルゴリズム，etc...)

Part 3. AI実践その1 ～ 画像認識モデルを作る

3-1. そもそもモデルって何？

機械学習では「モデル」を仮定し、そのモデルをいかに事前に用意したデータで調整するか、というところに本質があります。 ここで、モデルとは実際にはただの数式を表していて、これは自由に決めることが出来ます。

例えば株価予測を行う場合であれば、$f(x ::= \mathrm{今日の終わり値}) = ax + b$ のような数式もモデルの一例です。 この場合、$a$ と $b$ は未知ですが、この未知の部分を事前に用意した大量のデータから決定してしまおうということが、 機械学習に共通して言える性質です。

数式 ＝ モデルを複雑に仮定すれば、そのぶん未知の部分が多くなります。 これに対して、未知の部分を決定する方法 (機械学習アルゴリズム) を取り替えたり、 データをより多く用意したり、コンピュータの性能を上げてみたり、といったことを行います。

3-2. Pythonで画像認識モデルを作成し、学習させてみよう

ここではAIの中でも機械学習 (教師あり学習) の部分に着目し、 ディープラーニングのアプローチで分類問題を解くAIを実装してみます。 想定する問題は「手書きの数字を自動で識別してみる」というものです。

ディープラーニングとは言っても、いきなり数100にも積み重なったニューラルネットワークを実装/学習することは、非常に困難です。 今回は、5層程度のシンプルなニューラルネットワークを用いて、数字を認識してみます：

プログラムから他のプログラムを使用する

命令を組み合わせることで、Pythonにおいてさまざまなプログラムを作成することができます。 しかしながら、1から全てを自分で決め構築することは大変です。 そこで、他の人が作成したプログラムを使用することを考えます。

プログラミング言語には、それを使用する人たちを中心にコミュニティが存在し、 そこでの成果を誰でも享受できるような取り組みが行われています。 (これを特に、オープンソースコミュニティと言うこともあります。)

本ワークショップでは、

  • keras: Python用の機械学習プログラムをまとめたもの
  • matplotlib: Python用のグラフ描画プログラムをまとめたもの

以上2点を使用します。では、プログラムを読み込む命令を実行してみましょう：

import keras                               # 1
from keras.datasets.mnist import load_data # 2
from matplotlib.pyplot import imshow       # 3

それぞれの行はどのような意味でしょうか？ まず、keras は以下のような階層構造でプログラムを格納しています：

  • keras: 大カテゴリー
    • datasets: 小カテゴリー，機械学習用のデータをまとめたもの
      • mnist: 数字画像に関する機械学習用データをまとめたもの
        • load_data: データを読み込む命令
      • etc...
    • etc...

また、matplotlib は以下のような階層構造でプログラムを格納しています：

  • matplotlib: 大カテゴリー
    • pyplot: 小カテゴリー，描画に関する命令をまとめたもの
      • imshow: 画像を描画する命令
      • etc...
    • etc...

1行目は、karas というカテゴリーそのものを読み込んでいます。
2行目は、keras -> datasets -> mnist とカテゴリーをたどり、load_data というプログラム (命令) を読み込んでいます。
3行目は、matplotlib -> pyplot とカテゴリーをたどり、imshow というプログラム (命令) を読み込んでいます。

このように、Pythonでは階層構造を持つデータを扱うことができます。 これは、階層の名前を . (ドット) で繋いでいくことでアクセスすることができます。

数字画像に関する機械学習用データ

まず、以下のように命令を実行してみましょう：

(x_train, y_train), (x_test, y_test) = load_data()

これにより：

  1. load_data 命令が実行され、数字画像に関する機械学習用データが読み込まれる
  2. 1) で読み込んだデータを、x_train, y_train, x_test, y_test という記号にそれぞれ割り当てる

という処理が行われます。ここで：

  • x_?: 特徴ベクトル
  • y_?: 属性

上記をそれぞれ表しています。

「特徴ベクトル」とは、そのデータが何であるかを判断する材料のことです。 また「属性」とは、そのデータが何であるか (正解) を示すものです。

教師あり学習を行う上で、特徴ベクトルと属性の決め方は非常に重要です。 例えば、「男性と女性を見分ける」というタスクを機械学習で行うとき：

  1. (身長, 体重) の組を「特徴ベクトル」とする場合
  2. (のどぼとけの凹凸) を「特徴ベクトル」とする場合

上記それぞれでは、まったく正解率が異なってくることが想像できるかと思います。

一方で、「肥満度を予想する」というタスクを機械学習で行う場合では、 明らかに 1) の特徴ベクトルの方が有用です。

ディープラーニングの革新的なポイントの1つは、本来人手で慎重に構成していた特徴ベクトルを、 ほとんど元のデータと変わらない状態で取り扱えるようになっていることです。

特徴ベクトルの確認方法

以下のように命令を実行してみましょう：

imshow(x_train[0].reshape((28, 28)), cmap='gray')

これにより、特徴ベクトルの0番目をモノクロ表示することができます。

ここで、[ と ] で囲まれたデータのことを「リスト」と呼びます。 リストは複数のデータをまとめて扱うためによく使用されます。 リストのデータに1つづつアクセスするには、データ名[番号] という表記を使います。

特徴ベクトル/属性の下処理

従来手法と比べて、属性ベクトルをほとんど元のデータと変わらない状態で取り扱えることがディープラーニングの強みではありますが、 ある程度の下処理を行ってあげた方が、認識率は格段に良くなります。

今回は：

  • 画像のトリミング・リサイズ (事前に実施済み)
  • 画像のモノクロ化 (事前に実施済み)
  • 正規化
  • 一次元化

上記の下処理を特徴ベクトルに対して行います。また、属性に関してもOne-Hot変換というものを行います。

さっそく、以下のように命令を実行してみましょう：

NUM_CLASSES = 10

x_train = x_train.reshape(60000, 784) # 一次元化
x_train = x_train.astype('float32')
x_train = x_train / 255               # 正規化

x_test  = x_test.reshape(10000, 784)  # 一次元化
x_test  = x_test.astype('float32')
x_test  = x_test / 255                # 正規化

y_train = y_train.astype('int32')
y_train = keras.utils.np_utils.to_categorical(y_train, NUM_CLASSES) # One-Hot変換

y_test  = y_test.astype('int32')
y_test  = keras.utils.np_utils.to_categorical(y_test, NUM_CLASSES)  # One-Hot変換

ニューラルネットワーク・モデルの設計

Kerasでは、ニューラルネットワーク・モデルの定義を以下のように行います：

from keras.models import Sequential     # 1
from keras.layers import Dense, Dropout # 2
from keras.optimizers import RMSprop    # 3

# 4
model = Sequential()
model.add(Dense(512, activation='relu', input_shape=(784,)))
model.add(Dropout(0.2))
model.add(Dense(512, activation='relu'))
model.add(Dropout(0.2))
model.add(Dense(10, activation='softmax'))

model.compile(
    loss='categorical_crossentropy',
    optimizer=RMSprop(),
    metrics=['accuracy']
)

#1 は、keras -> models とカテゴリーをたどり、Sequential というプログラムを読み込んでいます。
#2 は、keras -> layers とカテゴリーをたどり、Dense ，Dropout というプログラムを読み込んでいます。
#3 は、keras -> optimizers とカテゴリーをたどり、RMSprop というプログラムを読み込んでいます。

#4 以降が、実際のモデルの定義部分です。今回は：

  1. 長さ784の特徴ベクトルを入力とし、活性化関数にLeRUを用いる、結合層 (ニューロン数は512)
  2. 20%の確率でニューロンを消去する、ドロップアウト層
  3. 活性化関数にLeRUを用いる、結合層 (ニューロン数は512)
  4. 20%の確率でニューロンを消去する、ドロップアウト層
  5. 活性化関数にSoftmaxを用いる、結合層 (ニューロン数は10)

===

  • 損失関数は交差エントロピーを元にする
  • 損失関数が最小となる点の探索を RMSProp によって行う

上記のようにモデルを定義しています。これにより、手書き画像を入力すると、 10個のニューロンに対してそれぞれ「画像がNである確率」が出力されることを期待します。

ニューラルネットワーク・モデルの学習

次に、以下のように命令を実行し、画像データから画像分類器を学習してみましょう：

BATCH_SIZE = 128
EPOCHS     = 20

history = model.fit(
    x_train, y_train,
    batch_size=BATCH_SIZE, epochs=EPOCHS,
    verbose=1, validation_data=(x_test, y_test)
)

ここで、

  • BATCH_SIZE: ニューラルネットワーク・モデルの更新を、データいくつごとに行うか
  • EPOCHS: 繰り返し学習させる回数

上記をそれぞれ表しています。x_train ，y_train を今回は学習用のデータとしています。 x_test ，y_test は学習結果の確認にのみ用い、学習には使用しません。

補足: なぜ学習に使用していないデータを、推論に使用するか？

教師あり学習の場合、事前に入力された学習用データの傾向を算出し、 それと比較して「未知の入力データはどの傾向になるか？」をもって推論を行います。

逆に言えば、事前に傾向を算出するために使用したデータは、 ほぼ確実にその傾向を示してしまうので、かなりの確率で正しい推論が行われてしまいます。

(x_train, y_train), (x_test, y_test) = load_data()

NUM_CLASSES = 10

x_train = x_train.reshape(60000, 784)
x_train = x_train.astype('float32')
x_train = x_train / 255

x_test  = x_test.reshape(10000, 784)
x_test  = x_test.astype('float32')
x_test  = x_test / 255

y_train = y_train.astype('int32')
y_train = keras.utils.np_utils.to_categorical(y_train, NUM_CLASSES)

y_test  = y_test.astype('int32')
y_test  = keras.utils.np_utils.to_categorical(y_test, NUM_CLASSES)

from keras.models import Sequential
from keras.layers import Dense, Dropout, Activation
from keras.optimizers import RMSprop

model = Sequential()
model.add(Dense(512, activation='relu', input_shape=(784,)))
model.add(Dropout(0.2))
model.add(Dense(512, activation='relu'))
model.add(Dropout(0.2))
model.add(Dense(10, activation='softmax'))

model.compile(
    loss='categorical_crossentropy',
    optimizer=RMSprop(),
    metrics=['accuracy']
)

BATCH_SIZE = 128
EPOCHS     = 20

history = model.fit(
    x_train, y_train,
    batch_size=BATCH_SIZE, epochs=EPOCHS,

Ctrl + C キーを同時に押してプログラムを止め、中身を確認してみましょう。次のコマンドを入力し、Enter キーを押します：

モータを制御するプログラムは以下の通りです：

# -*- coding: utf-8 -*-

__author__    = 'Kazuyuki TAKASE'
__copyright__ = 'PLEN Robotics Inc, and all authors'
__license__   = 'All Rights Reserved'


# 外部プログラムの読み込み
# =============================================================================
from wiringpi import *


# 定数定義・初期化処理
# =============================================================================
RIGHT_WHEEL_PIN = 12
LEFT_WHEEL_PIN  = 13

wiringPiSetupGpio()
pinMode(RIGHT_WHEEL_PIN, PWM_OUTPUT)
pinMode(LEFT_WHEEL_PIN, PWM_OUTPUT)
pwmSetMode(PWM_MODE_MS)
pwmSetRange(1024) # 1024 => 2^10 => 10[bit]に精度を設定
pwmSetClock(375)  # 基本周波数が50[Hz]となるように、基準周波数21.6[MHz]を分周
                  # { (21.6 x 10^6) / (1024 x 375) => 50[Hz] }


# 独自命令の定義
# =============================================================================
def forward():
    pwmWrite(RIGHT_WHEEL_PIN, 512) # 512 / 1024 => 50%出力
    pwmWrite(LEFT_WHEEL_PIN, 512)  # 512 / 1024 => 50%出力


def stop():
    pwmWrite(RIGHT_WHEEL_PIN, 0)   # 0 / 1024 => 0%出力
    pwmWrite(LEFT_WHEEL_PIN, 0)    # 0 / 1024 => 0%出力


def right():
    pwmWrite(RIGHT_WHEEL_PIN, 768) # 768 / 1024 => 75%出力
    pwmWrite(LEFT_WHEEL_PIN, 256)  # 256 / 1024 => 25%出力


def left():
    pwmWrite(RIGHT_WHEEL_PIN, 256) # 256 / 1024 => 25%出力
    pwmWrite(LEFT_WHEEL_PIN, 768)  # 768 / 1024 => 75%出力


# トップレベル・スクリプトの定義
# =============================================================================
if __name__ == '__main__':
    from time import sleep

    while True:
        forward(); sleep(2)
        stop();    sleep(2)

        right();   sleep(2)
        stop();    sleep(2)

        left();    sleep(2)
        stop();    sleep(2)

4-2. 交通信号を画像認識して、モータを制御してみよう

次に、交通信号を画像認識してモータを制御するだけのプログラムを動かしてみましょう。次のコマンドを入力し、Enter キーを押します：

sudo python3 main-signal.py

このプログラムを実行することにより、信号機の画像をカメラに見せることで「前進・停止」の動作が切り替わります。

Ctrl + C キーを同時に押してプログラムを止め、中身を確認してみましょう。次のコマンドを入力し、Enter キーを押します：

vim main-signal.py

モータを制御するプログラムは以下の通りです：

# -*- coding: utf-8 -*-

__author__    = 'Kazuyuki TAKASE'
__copyright__ = 'PLEN Robotics Inc, and all authors'
__license__   = 'All Rights Reserved'


from time import sleep

from mvnc import mvncapi as mvnc
import numpy as np
import cv2

from motor_driver import forward, stop


# Module Level Instances / Constants
# =============================================================================
ROOT_PATH       = '/home/pi/iot-04/'
GRAPH_FILE      = 'graph'
CATEGORIES_FILE = 'categories.txt'
INPUTSIZE_FILE  = 'inputsize.txt'


devices = mvnc.EnumerateDevices()

assert len(devices) != 0, 'Some Myriad device should be connected with the Raspberry Pi!'

device = mvnc.Device(devices[0])
device.OpenDevice()

with open(ROOT_PATH + GRAPH_FILE, mode='rb') as fin:
    graph = device.AllocateGraph(fin.read())

with open(ROOT_PATH + CATEGORIES_FILE, 'r') as fin:
    categories = list(filter(
        lambda x: x != 'classes',
        ( line.split('\n')[0] for line in fin.readlines() )
    ))

with open(ROOT_PATH + INPUTSIZE_FILE, 'r') as fin:
    image_size = int(fin.readline().split('\n')[0])

camera = cv2.VideoCapture(0)


# Original Functions
# =============================================================================
def square_trim(image):
    x, y, z = image.shape
    delta   = abs(x - y)

    if x > y:
        image = image[int(0.5 * delta): x - int(0.5 * delta), 0: y]

    else:
        image = image[0: x, int(0.5 * delta): y - int(0.5 * delta)]

    return cv2.resize(image, (image_size, image_size))


def normalize(image):
    for x in range(3):
        image[:, :, x] = (image[:, :, x] - 128) / 128

    return image


# Application Entry Point
# =============================================================================
while True:
    [ camera.read() for _ in range(4) ]

    _, image = camera.read()

    image = square_trim(image)
    image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
    image = normalize(image.astype(np.float16))

    graph.LoadTensor(image.astype(np.float16), 'user object')
    output, user_obj = graph.GetResult()

    top_accuracies = output.argsort()[: : -1][: 20]

    if any( ('traffic light' in categories[x]) for x in top_accuracies ):
        print('[STOP]')
        stop()

    else:
        print('[MOVE FORWARD]')
        forward()

    sleep(0.5)

4-2. で確認したプログラムでは、以下の行で「どの画像を識別するか？」を判断していました：

if any( ('traffic light' in categories[x]) for x in top_accuracies ):

ここで、traffic light という記述が何を識別したいかを決めている部分のため、 事前に学習済みのデータの中に識別を行いたいものが存在すれば、 この記述を変えるだけで自動運転カーの挙動を変えることができます。
